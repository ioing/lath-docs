DefaultSlideViewApplets

AppletConfig > defaultSlideViewApplets Snap 切换效果的程序坞，我们称之为 SlideView。

Types

 type SlideViewApplets = Array<{
              id: string,
              activate: 'lazy' | 'instant' | 'passive'
            }>
            interface AppletBaseConfig {
              defaultSlideViewApplets?: SlideViewApplets
            }

Details

“id”表示要被加载的程序卡名

“activate”表示程序卡的激活类型，共分为以下三种：

      <ul>
        <li>lazy: 在浏览器空闲时加载 Applet 页面内容。</li>
        <li>instant: 即刻加载 Applet 页面内容。</li>
        <li>passive: 在被激活时加载 Applet 页面内容。</li>
      </ul>

Example

 createApplication({
            applets: {
              frameworks: {
                config: {
                  defaultSlideViewApplets: [
                    {
                      id: "appletA",
                      activate: 'instant'
                    },
                    {
                      id: "appletB",
                      activate: 'lazy'
                    }
                  ]
                }
              }
            }
          })

Notes

设定 defaultSlideViewApplets 的程序卡将不再具有自身的视图（frameworks Applet 除外），而是其中嵌套了其它的程序卡视图，因此该配置与[source]相冲突，不应同时设置。

另外使用“API-Applet.slide.to”可使程序坞滑动到到指定程序卡位置。

OpenSlideViewLeftHolder

AppletConfig > openSlideViewLeftHolder 开启后，在 SlideView 效果下（defaultSlideViewApplets 配置时），最左侧边缘滑动时会允许退出应用。

该配置主要用于解除在首条历史记录下的退出受阻，若浏览历史记录大于 1 时则大多情况下不存在边缘滑动受阻的情况，若不开启则，在 ios 设备中可能出现边缘滑动时仅会触发边缘弹性，而不会回退到上一级。

Types

 interface AppletBaseConfig {
              openSlideViewLeftHolder?: boolean
            }

Details

该值默认为 false。

Example

 createApplication({
              applets: {
                home: {
                  config: {
                    defaultSlideViewApplets: [
                      {
                        id: "appletA",
                        activate: 'instant'
                      },
                      {
                        id: "appletB",
                        activate: 'lazy'
                      }
                    ],
                    openSlideViewLeftHolder: true
                  }
                }
              }
            })

Notes

开启该选项后需要注意程序卡左侧边缘将产生 20px 的无反馈区域，开发时应注意预留该安全操作距离。
